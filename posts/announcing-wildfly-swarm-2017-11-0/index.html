    WildFly Swarm 2017.11.0

    In Indiana, November is the month where your chances of having a car accident with a deer increase. It’s also the month where you can enjoy WildFly Swarm 2017.11.0.

    What’s New?

    Mostly documentation, which has moved to http://docs.wildfly-swarm.io/2017.11.0.

    Changelog

    Release notes for 2017.11.0 are available here.

    Bug

    • [SWARM-1108] Keycloak Server Customization impossible (Done)

    • [SWARM-1561] Reference Guide: keycloak-server documentation uses wrong property names (Done)

    • [SWARM-1590] project-defaults.yml not detected in hollow jar (Done)

    • [SWARM-1603] Readme documentation is not available for all Configuration samples (Done)

    • [SWARM-1610] Lack of consistency between the Reference for keycloak-server and the actual parameters (Done)

    • [SWARM-1612] Can’t set max-post-size on https-listeners (Done)

    • [SWARM-1615] Documentation link is incorrect on MicroProfile example (Done)

    • [SWARM-1617] License generation executing as part of javadoc and repo generation as well (Done)

    • [SWARM-1618] Typos in How To Create A Fraction Documentation (Done)

    Enhancement

    • [SWARM-1414] Adding timestamp to hollow jar prevents caching as Docker layer (Done)

    Task

    • [SWARM-1357] Reference uses swarm.messaging instead of swarm.messaging-activemq (Done)

    • [SWARM-1413] Update Project Stages doc in User Guide (Done)

    • [SWARM-1593] Update bom-certified version to 2017.10.1 (Done)

    Resources

    Per usual, we tend to hang out on irc.freenode.net in #wildfly-swarm.

    All bug and feature-tracking is kept in JIRA.

    Examples are available in https://github.com/wildfly-swarm/wildfly-swarm-examples/tree/2017.11.0
